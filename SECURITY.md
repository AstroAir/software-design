## 报告漏洞

如果你发现安全漏洞，请通过以下方式报告：

1. **不要**在公开的 Issue 中报告安全漏洞
2. 发送邮件至项目维护者
3. 提供漏洞的详细描述和复现步骤

我们会在收到报告后尽快回复，并在修复后发布安全更新。

## 安全最佳实践

### 密码存储

- 系统使用明文存储密码（仅供学习演示）
- 生产环境应使用加密存储

### 数据保护

- 数据以 JSON 格式存储在本地
- 建议定期备份数据文件
- 不要将包含敏感信息的数据文件提交到版本控制

### 权限控制

- 管理员密码应定期更换
- 不要使用默认密码
